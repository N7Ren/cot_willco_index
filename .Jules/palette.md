## 2026-03-05 - Custom Table Header Sorting Accessibility
**Learning:** Adding custom sorting functionality (`onclick="handleSort()"`) to `<th>` elements without native button elements renders the action inaccessible to keyboard and screen reader users. The table structure may naturally suggest focusability to a visual user but isn't interactive to accessibility tech.
**Action:** When creating custom interactive table headers for sorting, ensure to provide keyboard support by adding `tabindex="0"`, a keyboard event listener for Enter/Space (`onkeydown`), an explicit `role="columnheader"` (since adding `tabindex="0"` can sometimes strip default roles), and dynamic `aria-sort` ('ascending', 'descending', or 'none') attributes for state indication. Provide a visible focus state `th:focus-visible` for keyboard users and a `title` tooltip.
